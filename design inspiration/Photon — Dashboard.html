Spectrum / Get started

Send your first message

You're four steps from a working line. Pick a platform, scaffold the project, add your keys, then message your line to see it reply.

Step 01

Choose your first platform

Pick where your agent will send and receive messages first. The rest of this walkthrough — the scaffold command, the keys you paste, and the line you message — is tailored to whatever you choose here. You can add more platforms later.

Step 02

Scaffold the project

Run this with your package manager. It scaffolds a Spectrum project, installs everything, and leaves a ready .env at the root for the next step.

Step 03

Add your credentials

The scaffolder already left a .env at the root, so just paste your two keys in. The PROJECT_ID identifies the project and is safe to share. The PROJECT_SECRET authenticates your requests, so keep it private.

If the secret ever leaks, rotate it from the Configure page. The old one stops working the moment you generate a new one.

Step 04

Run it and text your line

Start the project with bun start and leave it running. Text the line below from your phone, and a second or two later you should get a reply back prefixed with echo:. If nothing comes back after a minute, the credentials from step three are usually the culprit.

iMessageShared

This is a shared number Photon assigned to your phone for test. It only replies to phones you've added, and yours was added at setup, which is why your text got through. Anyone else has to be added first.

Add usersThe people who'll text test

Want a number anyone can text without adding them first? The Business plan gives your project a dedicated number instead of one from the shared pool.

What's next

You're done. But you've barely started

You've got a working Spectrum project, but echoing text back is about as basic as it gets. The docs cover what's worth building on: multi-step conversations, tool calls, and shared state across platforms.
