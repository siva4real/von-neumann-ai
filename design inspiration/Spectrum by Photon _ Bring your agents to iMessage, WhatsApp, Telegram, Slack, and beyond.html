const spectrum = Spectrum(providers: [...], config: {...})
const agent = new Agent()

// Handle incoming messages by type
spectrum.onMessage(async ({ space, message }) => {
  switch (message.type) {
    case "text":
      const response = await agent.generate(message.content)
      await space.send("Hello World", attachment(...))
      await space.send(response)
    case "attachment":
      await message.react("like")
  }
})

// Auto-select the best available channel for delivery
await spectrum.send("+123456789", sticker(...))

// Explicitly route through iMessage with Telegram as fallback
await spectrum.send("+123456789", imessage().fallback(telegram()), poll({...}))

过去开发者都在想办法把交互界面做得更方便人类操作，去抢占用户的屏幕时间。现在的逻辑变了：开发者需要考虑的是，如何让软件和AI交互更顺畅，因为AI不会滑动屏幕，也不会点按钮。 人对AI发号施令的入口，也不再需要是一个新的App。用户对下载App已经很疲惫，新App第二天流失率往往高达70%。真正的入口，应该是人们每天高频使用的工具，比如iMessage、电话、社交软件，或者Terminal。 目前已经有产品在往这个方向走。比如郭宇把Agent放进了打电话的入口，解决了跨语种订餐的问题；硅谷团队 @photonhq 开发的Spectrum，则把Agent直接放在iMessage、WhatsApp、Telegram、电话和FaceTime的原生聊天框里，并且开放给所有应用接入。 这个新范式也改变了很多过去的习惯。以前用dating app时，大家匹配后还要交换联系方式，转到iMessage聊天。现在不一样了：在美国大学生圈子里比较火的Ditto，通过Spectrum的MCP，可以直接在iMessage里调用Ditto，填写个人信息后，Ditto就会发消息告诉你什么人对你感兴趣，整个过程都在iMessage里完成。 这样一来，开发一个dating app不再需要上百人的团队，也不用花重金在广告平台引导下载，更不用靠各种优惠或套路来留存用户。一个几人学生团队，花几天时间就能做出可能引爆市场的产品，而且不用担心留存问题，可以把精力真正放在产品运营上。 为什么GUI范式彻底过时了？  GUI的本质，是人类看不懂二进制，只能依赖软件厂商提供的“视觉翻译器”。过去20年，开发者都在疯狂卷界面好看度和操作便利性，结果用户APP疲劳严重，屏幕使用时间Top10永远是微信、iMessage这些老玩家。AI时代，真正的用户已经不是人类，而是AI Agent。它只会调用接口、执行命令、读输出。给它一个干净、可靠、可组合的CLI + Skill，它就能自动化、循环执行，还能和其他Agent顺畅协作。 CS新生应该立刻停止学习所有跟GUI相关的东西：iOS / Android开发、Web前端 / React / Flutter，以及任何以“让人类滑动屏幕”为目标的技术。  把时间花在如何给AI设计极致好用的CLI（低token、可管道、robust错误处理）、如何写高质量Skill（Markdown格式的SOP、触发条件、纠错逻辑），以及如何让Agent之间丝滑协作（A2A接口）。开发入口本身已经不重要了，重要的是让AI Agent觉得好用。 Spectrum正是这个新范式的底层基础设施 Spectrum把AI Agent直接“住进”用户每天都在用的原生聊天框： 用户发一条消息就能对话，无需下载App；单API打通iMessage+ 多平台； 专属手机号托管 + A2A能力； 智能过滤骚扰、自动执行任务，最后只推简洁报告。 这正是AI时代正确的入口逻辑，把Agent塞进人类屏幕使用时间Top1~Top3的通讯工具里，让好产品不再被下载壁垒卡死。
